# JavaScript Functions & Events


## 📚 Topics Covered

### 1. Functions
A function is a reusable block of code that performs a specific task. It helps reduce code repetition and makes programs easier to manage.

### 2. Why We Use Functions
Functions improve code reusability, readability, maintainability, and debugging. Instead of writing the same code multiple times, we can write it once and call it whenever needed.

### 3. Function Syntax
Learned how to create a function, pass parameters, and call it using arguments. Also understood the basic structure of a JavaScript function.

### 4. Parameters vs Arguments
Parameters are the placeholders defined in a function, while arguments are the actual values passed when calling the function.

### 5. Return Statement
The `return` keyword sends a value back from a function. Once a return statement executes, the function stops running.

### 6. Function vs Procedure
A function usually returns a value, whereas a procedure performs a task without returning anything. In JavaScript, both are written using the `function` keyword.

### 7. Pure vs Impure Functions
Pure functions always produce the same output for the same input and have no side effects. Impure functions can modify external data or depend on outside values.

### 8. Types of Functions
Learned three ways to create functions:
- Function Declaration
- Function Expression
- Arrow Function

### 9. Arrow Functions
Arrow functions provide a shorter and cleaner syntax for writing functions. They are commonly used for callbacks and simple operations.

### 10. Events
Events are actions performed by the user, such as clicking a button, typing in an input field, or moving the mouse.

### 11. addEventListener()
The `addEventListener()` method is used to attach an event handler to an HTML element, allowing the page to respond to user actions.

### 12. Event Object
The event object contains useful information about the triggered event, such as the target element and the event type.

### 13. Common Events
Learned commonly used JavaScript events such as:
- click
- dblclick
- input
- change
- submit
- keydown
- keyup
- mouseover
- mouseout
- load

### 14. Event Handling Pattern
Most interactive features follow a simple process:
1. Select the element.
2. Listen for an event.
3. Perform an action.

### 15. removeEventListener()
This method removes an event listener from an element. The same function reference must be used to successfully remove the listener.

### 16. Graceful Degradation
A webpage should still provide its basic content and functionality even if JavaScript is disabled.

- Write reusable and maintainable code
- Use different types of functions
- Handle user interactions with events
- Work with DOM event listeners
- Build interactive web pages
